# Архитектура и принцип работы

## Компоненты

- `cmd/cli` — интерактивный интерфейс установки, удаления и выбора профилей.
- `cmd/service` — IFEO debugger, фактическая точка входа при запуске Java-процесса игры.
- `cmd/metrics-helper` — утилита генерации `game_metrics.jsonl`.
- `internal/config` — загрузка/генерация конфигов.
- `internal/jvm` — формирование и фильтрация JVM-флагов.
- `internal/telemetry` — сбор процессных и игровых метрик.
- `internal/presetbench` — ранжирование пресетов и автовыбор.

## Поток запуска

1. `cli.exe --install` регистрирует IFEO-перехват.
2. Игра инициирует запуск Java runtime.
3. Windows вызывает `service.exe`.
4. `service.exe`:
   - определяет, это игровой запуск или bootstrap;
   - подгружает активный профиль;
   - применяет совместимые флаги;
   - стартует дочерний процесс;
   - собирает телеметрию до завершения.
5. После завершения:
   - пишется `wrapper.log`;
   - пишется JSONL-событие в `logs/presets/<preset>.jsonl`.

## Безопасность логов

- raw JVM args не логируются;
- чувствительные пути редактируются до `<user>`;
- размер `wrapper.log` ограничен.

## Поведение benchmark

- требуется минимум 2 успешных запуска на пресет;
- при наличии игровых метрик (`fps/frame_time`) они в приоритете;
- без игровых метрик используется soft fallback (`process cpu` + `wait_ms`);
- результат содержит confidence (`high/medium/low`).
