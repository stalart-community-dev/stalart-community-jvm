# Архитектура и принцип работы

## Компоненты

- `cmd/cli` — интерактивный интерфейс установки, удаления и выбора профилей.
- `cmd/service` — IFEO debugger, фактическая точка входа при запуске Java-процесса игры.
- `internal/config` — загрузка/генерация конфигов.
- `internal/jvm` — формирование и фильтрация JVM-флагов.

## Поток запуска

1. `cli.exe --install` регистрирует IFEO-перехват.
2. Игра инициирует запуск Java runtime.
3. Windows вызывает `service.exe`.
4. `service.exe`:
   - определяет, это игровой запуск или bootstrap;
   - подгружает активный профиль;
   - применяет совместимые флаги;
   - стартует дочерний процесс и ждет завершения.
5. После завершения:
   - пишется `wrapper.log`.

## Безопасность логов

- raw JVM args не логируются;
- чувствительные пути редактируются до `<user>`;
- размер `wrapper.log` ограничен.

## Поведение автоподбора

- при первичной инициализации выбирается рекомендованный пресет по железу;
- default-профиль ориентирован на `balanced` как базовый вариант;
- для low-end применяется `compat`, для high-end возможен `performance`/`ultra`.
